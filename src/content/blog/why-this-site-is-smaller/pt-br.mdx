## O formato antigo

Meu site anterior cresceu até se tornar um produto. Ele tinha contas, bancos de dados, dashboards, métricas, estado no cliente e infraestrutura suficiente para fazer a publicação de uma nota curta parecer a manutenção de uma aplicação.

Nenhuma dessas ferramentas é ruim por natureza. Elas apenas resolviam problemas que este site não tem mais.

## A nova restrição

Esta versão começa com uma pergunta mais rigorosa: do que um espaço pessoal na web realmente precisa?

Para mim, a resposta é um pequeno site estático, arquivos que posso ler no Git e páginas que continuam úteis sem uma aplicação no navegador. Astro renderiza a estrutura, MDX guarda os textos e CSS carrega quase toda a identidade visual.

A restrição não existe por minimalismo. Ela mantém a superfície operacional proporcional ao propósito do site.

## O que permanece

As partes importantes continuam aqui:

- textos longos com espaço para incerteza
- notas práticas com contexto suficiente para serem reutilizadas com segurança
- projetos selecionados sem transformá-los em material de venda
- traduções armazenadas como arquivos comuns
- formas diretas de entrar em contato

Não existe banco de dados para manter vivo nem perfil de visitante para montar. Publicar significa editar um arquivo, validá-lo e implantar uma saída estática.

## Um padrão mais tranquilo

Software tende a acumular recursos. Um site pessoal pode escolher outra direção.

Ele pode ser compreensível. Pode carregar rápido. Pode respeitar a atenção. Acima de tudo, pode parecer um lugar mantido por uma pessoa, e não um funil operado por um produto.
